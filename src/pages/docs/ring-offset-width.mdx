## Basic usage

### Setting the ring offset width

Use the `ring-offset-{width}` utilities to simulate an offset by adding solid white box-shadow and increasing the thickness of the accompanying outline ring to accommodate the offset.

<Example>

### Changing the offset color

You can't actually offset a box-shadow in CSS, so we have to fake it using a solid color shadow that matches the parent background color. We use white by default, but if you are adding a ring offset over a different background color, you should use the `ring-offset-{color}` utilities to match the parent background color:

<Example>
  <div class="grid place-content-center text-white text-sm text-center font-bold leading-6 text-white">
    <div class="flex flex-col items-center shrink-0">
      <p class="font-medium text-sm font-mono text-gray-500 text-center mb-3">ring-offset-pink-500</p>
      <button class="px-4 py-2 font-semibold text-sm bg-white text-gray-700 border border-gray-300 rounded-md shadow-sm ring-2 ring-offset-4 ring-offset-pink-100 ring-pink-500">Save Changes</button>
    </div>
  </div>
</Example>

```html
<button class="... ring ring-pink-500 **ring-offset-2** **ring-offset-pink-100**">
  Save Changes
</button>
```

For more information, see the [ring offset color](/docs/ring-offset-color) documentation.

## Using custom values

### Customizing your theme

To customize which ring offset width utilities are generated, add your custom values under `ringOffsetWidth` key in the `theme` section of your `tailwind.config.js` file.

```js tailwind.config.js
module.exports = {
  theme: {
    extend: {
      ringOffsetWidth: {
        '3': '3px',
        '6': '6px',
        '10': '10px',
      }
    }
  }
}
```

Learn more about customizing the default theme in the [theme customization](/docs/theme#customizing-the-default-theme) documentation.
